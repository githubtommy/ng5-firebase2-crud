Angular5 | Firebase CRUD


Fully Working Inventory-App Example by Tom Beaver


Overview

This is a simple but fully functioning Angular5/Firebase inventory app that implements full CRUD functionality. Users of this demo can try out all CRUD operations without having to set up their own Firebase account. Authentication is not required.

Although the main purpose is to show how to implement a Firebase backend into an Angular 6 app, this example also demonstrates many common solutions and patterns that most Angular 6 apps will require, such as routing, injection of global services, injection of dependencies at both the module and component levels, code organization, implementation and data-binding of various Bootstrap elements, and more.

Developers may grab this from GitHub: ng5-firebase2-crud and modify it as desired. They will need to have a Firebase account and apply their own Firebase APIKey and configuration properties, as explained in the README.




Main Technologies

Angular5, AngularFire2, Firebase Realtime Database, Angular CLI, Angular Router, Bootstrap 4, Angular Forms, NodeJS, TypeScript, ES6, FontAwesome Icons

UX/UI Features

  • Navigate between Home, List and Detail pages
  • List Page — View all records; sort and reverse-sort on any column; delete existing records; launch detail page to modify an existing record or create a new one
  • Detail Page — Modify fields of existing record; create a new record; apply a star rating to the record; save the new or modified record; clear the form; cancel and return to List page without saving changes

Angular Features

  • Created using Angular/CLI 1.6.1
  • Uses the latest Angular Routing approach
  • Follows best practices for code organization (organization by feature; shared components and services place at app level)
  • Globally required services injected at root-app level
  • Logging service implemented as an example of a global service
  • Uses a variety of Bootstrap elements and form controls
  • Uses FontAwesome icons
  • Firebase config and service implemented in a way that makes this an easy starting point for other Firebase-connected apps
  • Confirmation modal when deleting all records
